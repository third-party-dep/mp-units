glide_computer

This example presents the usage of:

  • quantity_point to mark “absolute” quantities like timestamp

  • quantity_point_kind to mark “absolute” kinds of quantities like altitude (as opposed to height),

  • different kinds for length, time, and speed,

  • the use of quantity kinds to show how to handle 3D space for aviation needs,

  • unit constants to initialize the database

  • quantities text output formatting,

  • cooperation with std::chrono.

45// An example of a really simplified tactical glide computer
46// Simplifications:
47// - glider 100% clean and with full factory performance (brand new painting)
48// - no influence of the ballast (pilot weight, water, etc) to glider performance
49// - only one point on a glider polar curve
50// - no influence of bank angle (during circling) on a glider performance
51// - no wind
52// - constant thermals strength
53// - thermals exactly where and when we need them ;-)
54// - no airspaces
55// - ground level changes linearly between waypoints
56// - no ground obstacles (i.e. mountains) to pass
57// - flight path exactly on a shortest possible line to destination

The use of quantity_kind and quantity_point_kind provides strong typing that divide quantities and quantity points to the ones on a horizontal (X, Y) plane and vertical (Z) axis. Their purpose is to make different kinds of quantity (i.e. length) separate strong types (i.e. distance, height). Additionally, separating a quantity from its quantity point means that we can define a height and altitude as different strong types. A quantity point provides a more restricted interface meant for absolute calculations.

The glide calculator engine also take benefits from different kinds of quantities. For example we have 3 for a quantity of length:

  • distance - a relative horizontal distance between 2 points on Earth

  • height - a relative altitude difference between 2 points in the air

  • altitude - an absolute altitude value measured form the mean sea level (AMSL).

The engine of this simple glide computer works on quantities of the SI units with a floating-point representation. The user can pass any unit valid for a given quantity and the library will automatically convert it to the one required by the engine interface.

The glide calculator takes task created as a list of waypoints, glider performance data, weather conditions, safety constraints, and a towing height.

 32task::legs task::make_legs(const waypoints& wpts)
 33{
 34  task::legs res;
 35  res.reserve(wpts.size() - 1);
 36  auto to_leg = [](const waypoint& w1, const waypoint& w2) { return task::leg(w1, w2); };
 37  std::ranges::transform(wpts.cbegin(), prev(wpts.cend()), next(wpts.cbegin()), wpts.cend(), std::back_inserter(res),
 38                         to_leg);
 39  return res;
 40}

 42std::vector<distance> task::make_leg_total_distances(const legs& legs)
 43{
 44  std::vector<distance> res;
 45  res.reserve(legs.size());
 46  auto to_length = [](const leg& l) { return l.get_length(); };
 47  std::transform_inclusive_scan(legs.cbegin(), legs.cend(), std::back_inserter(res), std::plus(), to_length);
 48  return res;
 49}

 51altitude terrain_level_alt(const task& t, const flight_point& pos)
 52{
 53  const task::leg& l = t.get_legs()[pos.leg_idx];
 54  const height alt_diff = l.end().alt - l.begin().alt;
 55  return l.begin().alt + alt_diff * ((pos.dist - t.get_leg_dist_offset(pos.leg_idx)) / l.get_length()).common();
 56}
 57
 58// Returns `x` of the intersection of a glide line and a terrain line.
 59// y = -x / glide_ratio + pos.alt;
 60// y = (finish_alt - ground_alt) / dist_to_finish * x + ground_alt + min_agl_height;
 61distance glide_distance(const flight_point& pos, const glider& g, const task& t, const safety& s, altitude ground_alt)
 62{
 63  const auto dist_to_finish = t.get_length() - pos.dist;
 64  return distance((ground_alt + s.min_agl_height - pos.alt).common() /
 65                  ((ground_alt - t.get_finish().alt) / dist_to_finish - 1 / glide_ratio(g.polar[0])));
 66}

 86flight_point tow(timestamp start_ts, const flight_point& pos, const aircraft_tow& at)
 87{
 88  const duration d = (at.height_agl / at.performance).common();
 89  const flight_point new_pos{pos.ts + d, pos.alt + at.height_agl, pos.leg_idx, pos.dist};

 95flight_point circle(timestamp start_ts, const flight_point& pos, const glider& g, const weather& w, const task& t,
 96                    height& height_to_gain)
 97{
 98  const height h_agl = agl(pos.alt, terrain_level_alt(t, pos));
 99  const height circling_height = std::min(w.cloud_base - h_agl, height_to_gain);
100  const rate_of_climb circling_rate = w.thermal_strength + g.polar[0].climb;
101  const duration d = (circling_height / circling_rate).common();
102  const flight_point new_pos{pos.ts + d, pos.alt + circling_height, pos.leg_idx, pos.dist};
103
104  height_to_gain -= circling_height;
105
106  print("Circle", start_ts, pos, new_pos);
107  return new_pos;
108}
109
110flight_point glide(timestamp start_ts, const flight_point& pos, const glider& g, const task& t, const safety& s)
111{
112  const auto ground_alt = terrain_level_alt(t, pos);
113  const auto dist = glide_distance(pos, g, t, s, ground_alt);
114  const auto new_distance = pos.dist + dist;
115  const auto alt = ground_alt + s.min_agl_height;
116  const auto l3d = length_3d(dist, pos.alt - alt);
117  const duration d = l3d / g.polar[0].v.common();
118  const flight_point new_pos{pos.ts + d, terrain_level_alt(t, pos) + s.min_agl_height, t.get_leg_index(new_distance),
119                             new_distance};
120
121  print("Glide", start_ts, pos, new_pos);
122  return new_pos;
123}

125flight_point final_glide(timestamp start_ts, const flight_point& pos, const glider& g, const task& t)
126{
127  const auto dist = t.get_length() - pos.dist;
128  const auto l3d = length_3d(dist, pos.alt - t.get_finish().alt);
129  const duration d = l3d / g.polar[0].v.common();
130  const flight_point new_pos{pos.ts + d, t.get_finish().alt, t.get_legs().size() - 1, pos.dist + dist};

132  print("Final Glide", start_ts, pos, new_pos);
133  return new_pos;

140void estimate(timestamp start_ts, const glider& g, const weather& w, const task& t, const safety& s,
141              const aircraft_tow& at)
142{
143  std::cout << STD_FMT::format("| {:<12} | {:^28} | {:^26} | {:^21} |\n", "Flight phase", "Duration", "Distance",
144                               "Height");
145  std::cout << STD_FMT::format("|{0:-^14}|{0:-^30}|{0:-^28}|{0:-^23}|\n", "");
146
147  // ready to takeoff
148  flight_point pos = takeoff(start_ts, t);
149
150  // estimate aircraft towing
151  pos = tow(start_ts, pos, at);
152
153  // estimate the altitude needed to reach the finish line from this place
154  const altitude final_glide_alt = t.get_finish().alt + height(t.get_length().common() / glide_ratio(g.polar[0]));
155
156  // how much height we still need to gain in the thermalls to reach the destination?
157  height height_to_gain = final_glide_alt - pos.alt;
158
159  do {
160    // glide to the next thermall
161    pos = glide(start_ts, pos, g, t, s);
162
163    // circle in a thermall to gain height
164    pos = circle(start_ts, pos, g, w, t, height_to_gain);
165  } while (height_to_gain > height{});
166
